一、项目概述

AgentCube 是 Volcano 社区的子项目，为 Kubernetes 提供 AI Agent 工作负载的原生调度与生命周期管理。解决传统批处理/推理系统无法满足的 Agent 特有需求：间歇性活动、亚秒级响应延迟、跨会话状态持久化。

核心能力：极低延迟调度、有状态生命周期管理（智能休眠/恢复）、高密度资源利用（预热池 + 性能隔离）、命令式 API。

六、核心特性列表

#特性说明
1AgentRuntime CRD自定义 AI Agent 运行时资源，支持 PodTemplate 定义、会话超时、最大会话时长
2CodeInterpreter CRD代码解释器资源，支持预热池 (WarmPool)、PicoD 认证模式、沙箱模板
3Workload ManagerHTTP API 服务：创建/删除 Sandbox、TokenReview 认证、h2c HTTP/2、垃圾回收
4Router 反向代理基于会话的反向代理：自动创建会话、JWT 授权、并发控制(信号量)、最后活动时间追踪
5Redis/Valkey 会话存储双后端支持：Redis（默认）和 Valkey，ZSET 过期/活动索引，JSON 序列化
6PicoD 沙箱守护进程Sandbox 内执行引擎：命令执行(60s超时)、文件上传/下载/列表、RSA JWT 认证
7AgentD 空闲清理基于 controller-runtime 的 Sandbox 空闲检测与自动清理（15分钟超时）
8Python CLI (kubectl-agentcube)5 个子命令：pack(打包)、build(构建)、publish(发布)、invoke(调用)、status(状态)
9Python SDKCodeInterpreterClient + AgentRuntimeClient，支持上下文管理器，连接池
10Helm Chart 部署一键部署：Router + WorkloadManager + 可选 Volcano Scheduler，完整 RBAC
11Dify 插件集成Dify 平台代码解释器工具插件，支持会话复用
12CI/CD 管线12 个 GitHub Actions：构建、发布(ghcr.io)、E2E、lint、测试覆盖、代码生成校验等
13SandboxWarmPool 预热CodeInterpreter 预创建 Sandbox 池，通过 SandboxClaim 快速分配
14JWT 身份认证Router ↔ PicoD 之间的 RSA-2048 密钥对管理，自动创建 K8s Secret
15多架构镜像构建Dockerfile 支持 CGO_ENABLED=0 + TARGETARCH/TARGETOS 多平台构建
