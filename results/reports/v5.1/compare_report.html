SDD-TEE v5.1 跨轮次对比报告 2026-04-12 11:08 UTC

对比轮次: 6 | 目标项目: AgentCube

执行摘要 (Executive Summary)

一、项目背景

本次评测的目标项目为 AgentCube， 一个为 Kubernetes 提供 AI Agent 工作负载原生调度与生命周期管理的云原生项目。项目涉及 Go 控制面、Python SDK/CLI、 Helm 部署、Dify 插件集成等多维度工程。
评测基于 SDD-TEE v5.1 的 8 阶段工作流（ST-0 ~ ST-7），将项目拆解为 43 个 AR， 旨在量化不同 AI Coding Assistant 在真实开源项目上的 Token 效率。

二、6 轮测试实录

  • 轮次 1 (opencode-cli / alibaba-cn/qwen3.6-plus): 生成了 371,798 行代码，耗时 26h 28m，消耗 196,174,321 Tokens，总成本 $101.03。 2 个 stage 无数据
  • 轮次 2 (opencode-cli / bailian-coding-plan/MiniMax-M2.5): 生成了 436,516 行代码，耗时 13h 45m，消耗 20,166,708 Tokens，总成本 $12.42。
  • 轮次 3 (opencode-cli / bailian-coding-plan/glm-4.7): 生成了 624,121 行代码，耗时 17h 16m，消耗 39,067,326 Tokens，总成本 $23.55。
  • 轮次 4 (opencode-cli / bailian-coding-plan/glm-5): 生成了 386,384 行代码，耗时 19h 4m，消耗 31,653,054 Tokens，总成本 $18.77。
  • 轮次 5 (opencode-cli / bailian-coding-plan/kimi-k2.5): 生成了 435,484 行代码，耗时 28h 35m，消耗 3,537,809 Tokens，总成本 $5.16。 64 个 stage 无数据
  • 轮次 6 (opencode-cli / bailian-coding-plan/qwen3.5-plus): 生成了 517,891 行代码，耗时 15h 48m，消耗 1,842,676 Tokens，总成本 $3.66。

三、核心指标极值

  • 产出规模最大: opencode-cli / bailian-coding-plan/glm-4.7 输出了 624,121 LOC 和 2883 个文件。
  • 经济性最优: opencode-cli / bailian-coding-plan/qwen3.5-plus 总成本仅 $3.66。
  • 交付速度最快: opencode-cli / bailian-coding-plan/MiniMax-M2.5 仅耗时 13h 45m 完成全量需求。
  • Token 效率最高: opencode-cli / bailian-coding-plan/qwen3.5-plus 每行代码平均消耗 3.6 Tokens/LOC (含缓存)，代码逻辑密度最高。
  • Cache 命中率最高: opencode-cli / bailian-coding-plan/kimi-k2.5 上下文复用率达到 79.9%。

洞察与建议：长上下文缓存命中率是降低多轮迭代成本的关键。不同模型在"代码骨架搭建"与"逻辑细节深挖"上的侧重点差异明显。 国产开放模型在生成规模和速度上已具备挑战业界顶尖基准的实力。

核心指标指南:
ET-LOC净 Token (input_net + output) / 生成代码行数 (LOC)。越低越好，但不包含 cache_read，在 cache 命中率高时不代表实际 API 流量。
ET-LOC-GROSS总 Token (input_net + cache_read + output) / 生成代码行数 (LOC)。越低越好，反映实际 API 流量 per LOC，是跨模型比较的公平指标。
Output 占比 (Gross%)Output Token / 总 Token (含缓存)。越低越表示上下文利用率高，代码生成任务通常在 1-15% 区间。
RT-RATIO人工输入 Token / AI 生成 Token。越低越好，代表高度自动化，AI 在无人工干预下完成任务的能力强。
Cache 命中率Cache Read / (Input + Cache Read)。越高越好，代表对长上下文的利用极其高效，大幅降低重复输入成本。
一致性评分跨模块/文件接口调用的一致性。越高越好，代表模型对复杂工程架构的整体把控能力。
代码可用率通过编译/静态检查的代码占比。越高越好，代表生成的代码具有实际生产价值。
