对比轮次: 5 | 目标项目: AgentCube

执行摘要 (Executive Summary)

一、项目背景

本次评测的目标项目为 AgentCube， 一个为 Kubernetes 提供 AI Agent 工作负载原生调度与生命周期管理的云原生项目。项目涉及 Go 控制面、Python SDK/CLI、 Helm 部署、Dify 插件集成等多维度工程。
评测基于 SDD-TEE v5.1 的 8 阶段工作流（ST-0 ~ ST-7），将项目拆解为 43 个 AR， 旨在量化不同 AI Coding Assistant 在真实开源项目上的 Token 效率。

二、5 轮测试实录

  • 轮次 1 (opencode-cli / alibaba-cn/qwen3.6-plus): 生成了 371,798 行代码，耗时 26h 28m，消耗 79,892,063 Tokens，总成本 $42.11。 51 个 stage 无数据
  • 轮次 2 (opencode-cli / bailian-coding-plan/MiniMax-M2.5): 生成了 436,516 行代码，耗时 13h 45m，消耗 20,166,708 Tokens，总成本 $18.29。
  • 轮次 3 (opencode-cli / bailian-coding-plan/glm-4.7): 生成了 624,121 行代码，耗时 17h 16m，消耗 39,067,326 Tokens，总成本 $23.89。
  • 轮次 4 (opencode-cli / bailian-coding-plan/glm-5): 生成了 386,384 行代码，耗时 19h 4m，消耗 31,653,054 Tokens，总成本 $15.26。
  • 轮次 5 (opencode-cli / bailian-coding-plan/qwen3.5-plus): 生成了 517,891 行代码，耗时 15h 48m，消耗 1,489,363 Tokens，总成本 $16.10。 35 个 stage 无数据

三、核心指标极值

  • 产出规模最大: opencode-cli / bailian-coding-plan/glm-4.7 输出了 624,121 LOC 和 2883 个文件。
  • 经济性最优: opencode-cli / bailian-coding-plan/glm-5 总成本仅 $15.26。
  • 交付速度最快: opencode-cli / bailian-coding-plan/MiniMax-M2.5 仅耗时 13h 45m 完成全量需求。
  • Token 效率最高: opencode-cli / bailian-coding-plan/qwen3.5-plus 每行代码平均消耗 4.4 Tokens/LOC，代码逻辑密度最高。
  • Cache 命中率最高: opencode-cli / bailian-coding-plan/qwen3.5-plus 上下文复用率达到 100.0%。

洞察与建议：长上下文缓存命中率是降低多轮迭代成本的关键。不同模型在"代码骨架搭建"与"逻辑细节深挖"上的侧重点差异明显。 国产开放模型在生成规模和速度上已具备挑战业界顶尖基准的实力。
