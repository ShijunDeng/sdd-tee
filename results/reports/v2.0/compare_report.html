SDD-TEE 跨轮次对比报告

生成时间: 2026-03-22 15:09 UTC | 对比轮次: 4 | 目标项目: agentcube

执行摘要 (Executive Summary)

一、项目背景

本次评测的目标项目为 AgentCube，一个复杂的云原生 AI Agent 工作负载管理平台。项目涉及 Kubernetes CRD 设计、Go 语言核心控制面、Python SDK、CLI 工具以及前/后端集成的多维度代码工程。
评测基于严谨的 CodeSpec 7 阶段工作流，将原始项目无损拆解为 43 个核心架构需求 (AR)，旨在考察不同 AI Coding Assistant 还原复杂系统、保持长上下文架构一致性的真实能力。

二、4 轮测试实录

  • 轮次 1 (gemini-cli / gemini-3.1-pro-preview): 生成了 46,754 行代码，耗时 58 分钟，消耗 1,743,157 Tokens，总成本 $42.13。
  • 轮次 2 (opencode-cli / bailian-coding-plan/kimi-k2.5): 生成了 11,235 行代码，耗时 121 分钟，消耗 603,468 Tokens，总成本 $12.32。
  • 轮次 3 (opencode-cli / bailian-coding-plan/qwen3.5-plus): 生成了 16,992 行代码，耗时 98 分钟，消耗 805,130 Tokens，总成本 $17.53。
  • 轮次 4 (opencode-cli / opencode/minimax-m2.5-free): 生成了 8,080 行代码，耗时 88 分钟，消耗 542,904 Tokens，总成本 $10.51。

三、核心指标极值

  • 行业基准 (Baseline): gemini-cli / gemini-3.1-pro-preview 展现了极度均衡的实力，生成 46,754 LOC，耗时 58 分钟，是所有模型对比的基石标杆。
  • 产出能力最优: gemini-cli / gemini-3.1-pro-preview 输出了最大的有效代码规模 (46,754 LOC)，在架构细节还原度上甚至超越了基准。
  • 交付速度最快: gemini-cli / gemini-3.1-pro-preview 仅耗时 58 分钟即完成了全量需求。
  • 代码浓度最高: gemini-cli / gemini-3.1-pro-preview 生成每行代码的平均 Token 消耗最低，逻辑密度极高，废话最少。
  • 综合成本最低: opencode-cli / opencode/minimax-m2.5-free 的任务总账单仅为 $10.51，极具经济性。

洞察与建议：长上下文缓存命中率成为降低多轮迭代成本的关键。不同模型在“代码骨架搭建”与“逻辑细节深挖”上的侧重点差异明显。目前的国产开放模型在生成规模和速度上已具备挑战业界顶尖基准 (Claude) 的实力。
