SDD-TEE 跨轮次对比报告

生成时间: 2026-03-22 15:01 UTC | 对比轮次: 7 | 目标项目: agentcube

执行摘要 (Executive Summary)

一、项目背景

本次评测的目标项目为 AgentCube，一个复杂的云原生 AI Agent 工作负载管理平台。项目涉及 Kubernetes CRD 设计、Go 语言核心控制面、Python SDK、CLI 工具以及前/后端集成的多维度代码工程。
评测基于严谨的 CodeSpec 7 阶段工作流，将原始项目无损拆解为 43 个核心架构需求 (AR)，旨在考察不同 AI Coding Assistant 还原复杂系统、保持长上下文架构一致性的真实能力。

二、7 轮测试实录

  • 轮次 1 (cursor-cli / claude-4.6-opus-high-thinking): 生成了 19,115 行代码，耗时 93 分钟，消耗 896,885 Tokens，总成本 $19.92。
  • 轮次 2 (opencode-cli / bailian-coding-plan/glm-5): 生成了 18,099 行代码，耗时 105 分钟，消耗 852,964 Tokens，总成本 $18.68。
  • 轮次 3 (opencode-cli / bailian-coding-plan/kimi-k2.5): 生成了 11,525 行代码，耗时 102 分钟，消耗 613,651 Tokens，总成本 $12.50。
  • 轮次 4 (opencode-cli / bailian-coding-plan/kimi-k2.5): 生成了 10,511 行代码，耗时 103 分钟，消耗 583,080 Tokens，总成本 $11.70。
  • 轮次 5 (opencode-cli / bailian-coding-plan/kimi-k2.5): 生成了 11,018 行代码，耗时 102 分钟，消耗 598,365 Tokens，总成本 $12.11。
  • 轮次 6 (opencode-cli / bailian-coding-plan/qwen3.5-plus): 生成了 16,237 行代码，耗时 77 分钟，消耗 754,346 Tokens，总成本 $16.08。
  • 轮次 7 (opencode-cli / opencode/minimax-m2.5-free): 生成了 20,380 行代码，耗时 99 分钟，消耗 885,287 Tokens，总成本 $19.64。

三、核心指标极值

  • 行业基准 (Baseline): cursor-cli / claude-4.6-opus-high-thinking 展现了极度均衡的实力，生成 19,115 LOC，耗时 93 分钟，是所有模型对比的基石标杆。
  • 产出能力最优: opencode-cli / opencode/minimax-m2.5-free 输出了最大的有效代码规模 (20,380 LOC)，在架构细节还原度上甚至超越了基准。
  • 交付速度最快: opencode-cli / bailian-coding-plan/qwen3.5-plus 仅耗时 77 分钟即完成了全量需求。
  • 代码浓度最高: opencode-cli / opencode/minimax-m2.5-free 生成每行代码的平均 Token 消耗最低，逻辑密度极高，废话最少。
  • 综合成本最低: opencode-cli / bailian-coding-plan/kimi-k2.5 的任务总账单仅为 $11.70，极具经济性。

洞察与建议：长上下文缓存命中率成为降低多轮迭代成本的关键。不同模型在“代码骨架搭建”与“逻辑细节深挖”上的侧重点差异明显。目前的国产开放模型在生成规模和速度上已具备挑战业界顶尖基准 (Claude) 的实力。
