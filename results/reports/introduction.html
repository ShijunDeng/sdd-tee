1. 框架概述

SDD-TEE（SDD Token Efficiency Evaluation）是一个基于 规范驱动开发 (Specification-Driven Development) 的 AI Coding Assistant Token 效率评估框架。

核心思路：逆向解析真实开源项目 agentcube 的源码生成结构化规范文档，再用不同的 AI 编码工具（Claude Code、Aider、Cursor 等）× 不同模型组合， 端到端完成 SDD 开发，量化各阶段的 Token 消耗、耗时和代码质量，为后续 Token 提效研究提供评估基线。

仓库地址：https://github.com/ShijunDeng/sdd-tee

5. 评测框架技术架构

5.1 流水线阶段定义

前置工作（一次性，不计入评测基线）

编号阶段说明需 LLM
P-A项目技术解析分析目标项目的代码量、技术栈、模块结构、API 端点等，生成 HTML 技术报告✓
P-B规范逆向生成从源码逆向工程，生成结构化 OpenSpec 规范文档 (Markdown)✓

评测阶段（每轮执行）

编号阶段说明追踪 Token
S-0SDD 端到端开发基于规范驱动 AI 工具生成完整项目代码✓
S-1质量验证对比生成代码与原始项目的结构/规模/语法一致性—
S-2数据汇总聚合 token、耗时、质量指标到 JSON/CSV—
S-3报告生成生成详细 HTML 评测报告（中文）含 token 分阶段明细—

5.2 评测维度

维度指标
Token 消耗input / output / cache_read / cache_write tokens（按阶段细分）
成本USD / CNY（按阶段、按工具×模型组合）
耗时Wall-clock 秒（端到端 & 按阶段）
代码质量文件数比、LOC 比、目录相似度、文件重叠率、关键文件覆盖、语法通过率
内容效率有效代码比例、注释比例、样板代码比例、推理/思考 tokens 比例

5.3 Token 追踪方案

方式说明
LiteLLM Proxy统一代理层拦截所有 API 调用，工具无关，精确 per-request 记录
工具原生Claude Code (OpenTelemetry) / Aider (session cost) / Cursor (有限)

推荐策略：双轨追踪：LiteLLM Proxy (统一口径) + 工具原生 (交叉验证)
