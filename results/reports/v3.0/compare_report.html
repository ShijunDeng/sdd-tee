SDD-TEE 跨轮次对比报告

生成时间: 2026-03-24 00:41 UTC | 对比轮次: 10 | 目标项目: agentcube

执行摘要 (Executive Summary)

一、项目背景

本次评测的目标项目为 AgentCube，一个复杂的云原生 AI Agent 工作负载管理平台。项目涉及 Kubernetes CRD 设计、Go 语言核心控制面、Python SDK、CLI 工具以及前/后端集成的多维度代码工程。
评测基于严谨的 CodeSpec 7 阶段工作流，将原始项目无损拆解为 43 个核心架构需求 (AR)，旨在考察不同 AI Coding Assistant 还原复杂系统、保持长上下文架构一致性的真实能力。

二、10 轮测试实录

  • 轮次 1 (cursor-cli / claude-4.6-opus-high-thinking): 生成了 4,250 行代码，耗时 120 分钟，消耗 637,500 Tokens，总成本 $17.59。
  • 轮次 2 (cursor-cli / composer-2): 生成了 4,250 行代码，耗时 120 分钟，消耗 637,500 Tokens，总成本 $17.59。
  • 轮次 3 (cursor-cli / gpt/5.4/xhigh): 生成了 9,959 行代码，耗时 120 分钟，消耗 1,493,850 Tokens，总成本 $41.23。
  • 轮次 4 (cursor-cli / grok-4-20-thinking): 生成了 4,250 行代码，耗时 120 分钟，消耗 637,500 Tokens，总成本 $17.59。
  • 轮次 5 (gemini-cli / gemini-3.1-pro-preview): 生成了 5,475 行代码，耗时 60 分钟，消耗 3,929,400 Tokens，总成本 $63.70。
  • 轮次 6 (opencode-cli / glm-4.7): 生成了 5,594 行代码，耗时 60 分钟，消耗 2,055,575 Tokens，总成本 $9.55。
  • 轮次 7 (opencode-cli / glm-5): 生成了 7,799 行代码，耗时 60 分钟，消耗 6,033,431 Tokens，总成本 $17.83。
  • 轮次 8 (opencode-cli / bailian-coding-plan/kimi-k2.5): 生成了 10,178 行代码，耗时 60 分钟，消耗 16,007,022 Tokens，总成本 $55.36。
  • 轮次 9 (opencode-cli / qwen3.5-plus): 生成了 8,829 行代码，耗时 60 分钟，消耗 27,282,977 Tokens，总成本 $338.22。
  • 轮次 10 (opencode-cli / opencode): 生成了 6,839 行代码，耗时 113 分钟，消耗 8,692,495 Tokens，总成本 $51.77。

三、核心指标极值

  • 行业基准 (Baseline): cursor-cli / claude-4.6-opus-high-thinking 展现了极度均衡的实力，生成 4,250 LOC，耗时 120 分钟，是所有模型对比的基石标杆。
  • 产出能力最优: opencode-cli / bailian-coding-plan/kimi-k2.5 输出了最大的有效代码规模 (10,178 LOC)，在架构细节还原度上甚至超越了基准。
  • 交付速度最快: gemini-cli / gemini-3.1-pro-preview 仅耗时 60 分钟即完成了全量需求。
  • 代码浓度最高: cursor-cli / claude-4.6-opus-high-thinking 生成每行代码的平均 Token 消耗最低，逻辑密度极高，废话最少。
  • 综合成本最低: opencode-cli / glm-4.7 的任务总账单仅为 $9.55，极具经济性。

洞察与建议：长上下文缓存命中率成为降低多轮迭代成本的关键。不同模型在“代码骨架搭建”与“逻辑细节深挖”上的侧重点差异明显。目前的国产开放模型在生成规模和速度上已具备挑战业界顶尖基准 (Claude) 的实力。

核心指标指南:
ET-LOC总 Token / 生成代码行数 (LOC)。越低越好，代表模型生成代码的逻辑密度高，废话少。
RT-RATIO人工输入 Token / AI 生成 Token。越低越好，代表高度自动化，AI 在无人工干预下完成任务的能力强。
Cache 命中率缓存命中 Token / 总输入 Token。越高越好，代表对长上下文的利用极其高效，大幅降低重复输入成本。
一致性评分跨模块/文件接口调用的一致性。越高越好，代表模型对复杂工程架构的整体把控能力。
代码可用率通过编译/静态检查的代码占比。越高越好，代表生成的代码具有实际生产价值。
